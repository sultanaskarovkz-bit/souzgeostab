# souyzgeostab.kz

Многостраничный сайт ТОО «СоюзГеоСтаб». Замена одностраничнику на Tilda
по Договору № 116/2026 (Приложение № 1, Месяц 1).

Чистая статика: HTML + CSS + один JS-файл без зависимостей. На хостинге
ничего, кроме PHP для формы, не требуется.

## Что залить на хостинг

Всё содержимое репозитория, кроме папки `tools/`. То есть:

```
index.html  uslugi/  tehnologii/  otrasli/  kejsy/  o-kompanii/
dokumenty/  kontakty/  etapy-raboty/  404.html
assets/  files/  form.php  .htaccess  robots.txt  sitemap.xml
```

Папка `tools/` — только для разработки, на сервере не нужна.

## Структура

| Раздел | Что внутри | Логика |
|---|---|---|
| `/uslugi/` | 7 страниц | Язык клиента: «просадка фундамента», «вода в подвале» |
| `/tehnologii/` | 7 страниц | Язык технологии: «jet grouting», «струйная цементация» |
| `/otrasli/` | 8 страниц | Язык объекта: шахты, дороги, ГТС, ТЭК |
| Прочее | 6 страниц | Этапы, кейсы, документы, о компании, контакты, 404 |

Разделение услуг и технологий сделано намеренно: иначе страницы «Подъём плит»
и «Floor Lifting» конкурировали бы за один запрос, и Google выбрал бы одну.

## Правка

Итоговые `.html` — обычные файлы, правятся руками и заливаются на хостинг.

Для массовых правок (текст в шапке, подвале, меню, добавление страницы)
удобнее менять данные и пересобирать:

```bash
node tools/build.mjs
```

Контент лежит в `tools/content/`, контакты и меню — в `tools/site.mjs`.
Если генератор не нужен, папку `tools/` можно удалить: на работу сайта
это не влияет.

Обработка изображений из экспорта Tilda (нужен `npm i`):

```bash
node tools/images.mjs
```
